  4. What are lists?
    5. In HTML, lists are essential for organizing and presenting items or information clearly. They allow for the grouping of related data, which improves readability and the overall user experience on web pages. You commonly use lists for navigation menus.
    • Ordered List
      • Purpose: Displays items in a specific, numbered order.
    • Unordered List
      • Purpose: Shows items with bullets, order does not matter.
    • Description List
      • Purpose: Pairs terms with their descriptions (sometimes called definition lists).

  5. What are attributes?
    6. Attributes in HTML are special words or modifiers used within the opening tag of an HTML element to provide additional information about that element. They help adjust the element's behavior, appearance, or functionality in various ways. Each attribute consists of a name and a value.
    Key points about attributes:
    1. Attributes modify elements by adding characteristics or controls.
    2. They always appear in the start tag of an element.
    3. Each attribute has a name (like href, src, class, id) and a corresponding value.
    4. Attribute values are typically enclosed in quotes.

  6. What are elements?
    7. An HTML element is a component of an HTML document that tells a web browser how to structure and interpret a part of the HTML document. HTML elements can contain formatting instructions, semantic meaning, and content.

  7. What are difference between XML and HTML?

    Purpose Designed to store and transport data with a focus on data structure Designed to display and format data in a web browser
    Tag Definition Tags are user-defined and extensible Tags are predefined and fixed
    Case Sensitivity Case-sensitive (tags and attributes must match exactly) Not case-sensitive
    Tag Closure All tags must be properly closed Some tags can be left unclosed
